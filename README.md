# Campus-Helpdesk-and-Service-Portal
Campus Helpdesk and Service Portal is a web-based university support system that centralizes academic, technical, and administrative services into one platform. It integrates an AI chatbot, a categorized FAQ system, and a ticket management system to provide fast support, transparent tracking, and improved user experience for students and staff.

## Project Overview

In modern university environments, students and staff often struggle with fragmented support channels such as emails, phone calls, and in-person visits. These methods lead to delayed responses, repeated inquiries, and inefficient service handling.

The **Campus Helpdesk and Service Portal** addresses this problem by providing a centralized digital platform that combines:
- A categorized FAQ system
- An AI-powered chatbot
- A ticket submission and tracking system

This approach ensures faster responses, better transparency, and improved communication between users and support teams.

---

## Project Objectives

- Centralize all campus support services into one platform  
- Reduce response time using AI automation  
- Provide real-time assistance through chatbot and human agents  
- Enable transparent ticket tracking and status updates  
- Reduce repetitive inquiries using a structured FAQ system  
- Improve overall service quality and user satisfaction  

---

## Target Users

- **Students**  
  - Browse FAQs  
  - Interact with the AI chatbot  
  - Submit and track support tickets  
  - Provide feedback after resolution  

- **Staff / Support Agents**  
  - View and respond to tickets  
  - Update ticket status (Open, Pending, Closed)  

- **Administrators**  
  - Manage FAQs (add, edit, delete)  
  - Monitor ticket statistics  
  - Oversee system operations

## System Architecture

The system follows a **three-tier architecture**:

### Presentation Layer
- User web interface
- Admin dashboard
- Chatbot interface

### Application Layer
- Authentication & authorization
- FAQ management
- Ticket management
- AI chatbot processing
- Feedback handling

### Data Layer
- User database
- FAQ database
- Ticket database
- Feedback database

This architecture improves scalability, maintainability, and security.
